### 1.2 功能说明

**目标用户**：个人用户、小型商家试用

- 客户数量：最多 20 个

- ❌ 高级模板

**目标用户**：长期使用商家、中大型商家

**核心权益**：
- 月度会员全部权益

## 二、定价策略

### 2.1 定价逻辑

| 方案 | 月单价 | 年总价 | 相当于 |

| 月度会员 | ¥29.9 | ¥358.8 | 月付原价 |

### 2.2 转化策略

**免费版 → 月度会员**
- 触发场景：商品数接近 50 个上限
- 引导文案："已添加 45/50 个商品，升级会员解锁无限商品"
- 转化点：功能限制提示

**月度会员 → 年度会员**
- 触发场景：连续订阅 3 个月
- 引导文案："已连续订阅 3 个月，升级年付立省 ¥159"
- 转化点：价格优惠

### 2.3 促销活动（预留）

- 新用户首月：¥9.9（原价 ¥29.9）
- 年付折扣：立减 ¥100（实际 ¥99/年）

**推荐返利**
- 推荐好友购买会员

## 三、功能权限控制

### 3.1 权限检查流程

```
用户操作请求
    ↓
检查用户订阅状态
    ↓
获取用户会员等级
    ↓
查询权限配置（limits）
    ↓
验证操作是否允许
    ↓
执行操作 / 返回权限不足提示
```

### 3.2 权限配置表（limits 字段）

```typescript
interface MemberLimits {
  max_products: number      // 最大商品数（-1 = 无限）
  max_customers: number     // 最大客户数（-1 = 无限）
  max_quotes: number        // 最大报价单数（-1 = 无限）
  max_templates: number     // 最大模板数（-1 = 无限）
  has_advanced_templates: boolean  // 高级模板
  has_ad: boolean           // 是否展示广告
  has_export_pdf: boolean   // 导出 PDF
  has_data_statistics: boolean // 数据统计
}
```

### 3.3 权限检查示例

**添加商品时检查**：
```typescript
// 后端中间件
async function checkProductLimit(userId: string) {
  const subscription = await getUserSubscription(userId)
  const tier = await getMembershipTier(subscription.tier_id)
  
  if (tier.limits.max_products === -1) {
    return true // 无限商品
  }
  
  const currentCount = await getProductCount(userId)
  if (currentCount >= tier.limits.max_products) {
    throw new Error('商品数量已达上限，请升级会员')
  }
  
  return true
}
```

## 四、支付集成

### 4.1 微信支付流程

```
用户选择会员方案
    ↓
前端调用创建订单 API
    ↓
后端生成商户订单号
    ↓
调用微信支付统一下单接口
    ↓
返回支付参数给前端
    ↓
前端调起微信支付
    ↓
支付结果回调通知
    ↓
更新用户订阅状态
```

### 4.2 订单状态

| 状态 | 说明 |
|:---|:---|
| pending | 待支付 |
| success | 支付成功 |
| failed | 支付失败 |
| refunded | 已退款 |

### 4.3 支付安全

- 订单号唯一性检查
- 支付金额校验
- 回调签名验证
- 幂等性处理（避免重复处理）

## 五、订阅管理

### 5.1 订阅状态

| 状态 | 说明 |
|:---|:---|
| active | 订阅有效 |
| expired | 已过期 |
| cancelled | 已取消 |

### 5.2 订阅流程

**新订阅**：
1. 用户选择会员方案
2. 完成支付
3. 创建订阅记录（status=active, start_at=now, expire_at=now+duration）
4. 用户立即享受会员权益

**续费**：
1. 用户点击续费
2. 完成支付
3. 延长订阅时间（expire_at = max(old_expire_at, now) + duration）

**过期处理**：
1. 定时任务检查 expire_at
2. 过期用户自动降级为免费版
3. 发送过期提醒通知

### 5.3 自动续费（预留）

- 支付前勾选"自动续费"
- 到期前 3 天发送续费提醒
- 到期当天自动扣款续费
- 用户可随时关闭自动续费
